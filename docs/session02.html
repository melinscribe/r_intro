Exploring Data

🎯 Learning Goals

By the end of this session, you will be able to:

  • Load data from .csv files and R packages.
  • Explore the structure of a dataset (columns, rows, variable types).
  • Understand the working directory in RStudio.
  • Practice using UCDP and ACLED conflict datasets.

2. Working with Data

📂 What is a .csv?

  • CSV = Comma-Separated Values
  • A plain text file with data separated by commas:
country,year,fatalities
Kenya,2020,42
Sudan,2020,13
  • Can be opened by R, Python, Excel, Stata, SPSS…
  • Universally compatible for research.

🧩 Rows vs Columns

  • Each row = an observation (e.g., a conflict event)
  • Each column = a variable (e.g., country, year, fatalities)

Example:

country year event_type fatalities
Kenya 2020 Battle 42
Sudan 2020 Protest 13

📊 What About Excel Files?

  • Excel files (.xls / .xlsx) have:
    • Formatting (colors, bold, merged cells)
    • Formulas (like =SUM(A1:A10))
    • Multiple sheets
  • Excel is good for humans, but not ideal for code.

✅ Why We Prefer CSV in Research

  • Plain text → no formatting headaches
  • Small file size
  • Opens everywhere (R, Python, SQL, Stata)
  • Version control friendly (Git can track changes)

💡 In short: CSVs are cleaner, lighter, and easier to automate with R.

📥 Reading Data

The most common way to read a CSV file is with read.csv

Let’s try it with a sample UCDP dataset (replace with your path):

ucdp_drc <- read.csv("Data/ucdp_drc.csv", sep=";", stringsAsFactors = FALSE)

📜 What is stringsAsFactors?

In older versions of R (before R 4.0), whenever you loaded a CSV file or created a data frame, all text (strings) would automatically be converted into factors unless you told R not to.

  • Factor = a categorical variable (like “Country”: Kenya, Sudan, Somalia)
  • String = plain text

This caused problems, functions would fail or behave strangely because they received a factor instead of a string.

Since R 4.0 (April 2020), the default behavior changed. Now, R does not convert strings to factors by default.

👉 But you’ll still see stringsAsFactors = FALSE in code for compatibility and good habit.
