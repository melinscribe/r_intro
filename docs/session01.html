Getting Started with R

🎯 Learning Goals

By the end of this session, you will be able to:

  • Understand what R and RStudio are used for.
  • Run commands and get help.
  • Create and name objects.
  • Understand R’s basic data types: numeric, character, logical.

1. Introduction

Welcome to R for Conflict Research!

R is a statistical programming language widely used in academia, NGOs, and policy institutions.
In this course, we’ll use R with conflict datasets to learn data wrangling, visualization, and analysis. It’s based on the original script and is fully runnable — you can click Run on each code chunk.

The # is your comment character in R, like * in Stata.

Most important shortcuts

  • Execute code:

    • Ctrl + Enter (Windows)

    • Alt + Cmd + I (Mac)

  • Pipe (%>%):

🏁 Running Commands and Getting Help

In RStudio, you can run a command by placing your cursor on the line and pressing:

  • Mac: ⌘ + Enter
  • Windows: Ctrl + Enter

If you forget how a command works, type ? before it:

?sum
?rm

💡 Try this with any function you don’t understand, like ?mean or ?plot.

📝 Assignment and Objects

Every programming language needs an assignment operator.

In R, we use <- to “put” a value into an object.

a <- 1
a
[1] 1

➡️ Here we created an object a and assigned the number 1 to it.

Objects can be overwritten:

b <- 1 + 5
b
[1] 6
c <- b - a
c
[1] 5

📛 Naming Rules for Objects

R is picky about object names.

# 1ab <- 2   # ❌ error: cannot start with a number
# _a <- 2    # ❌ error: cannot start with underscore
.a <- 3      # ✅ works! variables can start with a dot

💡 Tip: Use snake_case (e.g. conflict_data) for readability.

🔢 Basic Data Types

R has several key data types:

  • numeric – numbers
  • character – text/strings
  • logical – TRUE or FALSE
num <- 42
char <- "conflict"
char2 <-"Strings can also be full sentences."
logic <- TRUE

Check their types:

class(num)
[1] "numeric"
class(char)
[1] "character"
class(logic)
[1] "logical"

📚 Vectors

Vectors are one-dimensional collections of the same type.

numbers <- c(1, 2, 3, 4)
words <- c("peace", "conflict", "war")

numbers
[1] 1 2 3 4
words
[1] "peace"    "conflict" "war"     
v1 <- c(1, 4, 10)
c(v1, 120)
[1]   1   4  10 120
c(120, v1)
[1] 120   1   4  10
v3 <- c(v1, "a")  # vector becomes character

Extract elements:

v1[1]             # first element
[1] 1
v1[c(1,3)]        # first & third element
[1]  1 10

Math is vectorized:

c(100, 200, 300) / c(10, 20, 30)
[1] 10 10 10

Modes

a1 <- c(1,4,9,17)
mode(a1)

a2 <- 1:9
mode(a2)

Type & Mode Errors

Each R command only works with appropriate object types.

sum(a1)   # works, because numeric
[1] 31

Error in a1 + a4: non-numeric argument to binary operator

A matrix is a two-dimensional array. Like vectors, it can take various data types: numeric, logical, character and, as we can see, all elements have the same data type.

If we want to combine two vectors (e.g. countries and years), we can do this with the cbind() command. This command combines columns whereas the rbind() command combines rows. Both commands create matrices.

countries <- c("France", "Germany", "Italy")
years <- c("2019", "1992", "2013")
combined <- cbind(countries, years)

We can now have a look at our combined object:

combined
     countries years 
[1,] "France"  "2019"
[2,] "Germany" "1992"
[3,] "Italy"   "2013"

Alternatively, we can use View() to inspect our data. The View() command (note, capitalization in the beginning is important!) is similar to the browse command in Stata.

# View(combined)

How to access values in a matrix?

To access a single value in a matrix, we need to think in a matrix. We need to call the matrix object first with mat_combined and then add with square brackets the rows, columns, or cells. The basic syntax is as follows [row, column]. If we just want to access the third row, we can write:

combined[3,] # We access the third row
countries     years 
  "Italy"    "2013"

If we want to access the second column:

combined[,2] # We access the second column
[1] "2019" "1992" "2013"

Data frames

Instead of creating matrices, we can also create a data frame.

df_combined <- data.frame(countries, years)

Data frames are a standard way of handling social science problems in R. We generate a data.frame using data.frame().

We add stringsAsFactors=FALSE to avoid that R transforms our string variable countries in factors.

df_combined <- data.frame(countries, years, stringsAsFactors = FALSE)

How to access parts of a data frame?

As in matrices, we can also access variables in data frames– that are usually listed in the columns of a data frame – with the dollar sign ($). Let’s assume we want to access the column that contains information on countries in the data frame df_combined:

df_combined$countries
[1] "France"  "Germany" "Italy"

Accessing single columns seems to be fairly simple. But what if you want to access the very first observation of a column? (This might be useful for later data wrangling exercises.) We will use functions from the dplyr package here.

# Load package
# install.packages("dplyr")
library(dplyr)

To access the first observation, we use first().

first(df_combined$countries)
[1] "France"

If we want to access the last observation, we use last().

last(df_combined$countries)
[1] "Italy"

Or, if we want to access the nth observation, we simply use nth(). Note, we need to specify which observation we want to display!

nth(df_combined$countries, 2)
[1] "Germany"

Bonus: What is the difference between a data.frame and a data.table?

A data.table is a different format of a data.frame that allows you to work with larger data sets.

2. Markdown and Quarto

📜 What is Markdown?

✅ Markdown is a lightweight markup language for formatting text.

  • Created in 2004 by John Gruber.
  • Uses plain text symbols for formatting:
    • # for headings
    • ** ** for bold
    • - for bullet points

💡 Markdown itself cannot run code — it’s just formatting.

📚 What is R Markdown?

✅ R Markdown = Markdown + R code.

  • Introduced by RStudio in 2014.
  • Allows you to mix:
    • ✍ Narrative text (Markdown)
    • 🔢 Code chunks (R)
    • 📊 Outputs (tables, plots)

✅ Can produce HTML reports, PDFs, Word docs, books, and slides.

🚀 What is Quarto?

✅ Quarto is the next generation of R Markdown.

  • Supports R, Python, Julia, and Observable JS in the same document.
  • Has a cleaner, more consistent system for:
    • 📄 Reports
    • 🎞 Slides
    • 📚 Books & Websites
  • Uses .qmd files instead of .Rmd.

💡 Think of Quarto as R Markdown 2.0 — more languages, more features.

🔍 Quarto vs R Markdown

Feature Markdown R Markdown Quarto
Text formatting ✅ ✅ ✅
R code chunks ❌ ✅ ✅
Multiple languages ❌ limited ✅
Slides/books ❌ ✅ ✅ (easier)

How to start an R Script

# Remove all objects from R's memory
rm(list=ls())  
# We will use the following code to install all packages
packages <- c("ggplot2")

# Install uninstalled packages
lapply(packages[!(packages %in% installed.packages())], install.packages)
list()
# Load all packages to library
lapply(packages, library, character.only = TRUE)
[[1]]
