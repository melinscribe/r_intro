üìà 1) What is Linear Regression?

Linear regression models a continuous outcome as a linear function of predictors.

General form:

\(Y = \beta_0 + \beta_1 X + \epsilon\)

  • \(Y\)end:math:text$ = outcome (e.g., number of fatalities)
  • \(X\)end:math:text$ = predictor (e.g., GDP per capita)
  • \(\beta_1\) = effect of GDP on fatalities
  • \(\epsilon\) = error term

üîç Model Diagnostics and Fit

When we estimate regressions, we need to check how well the model fits the data and whether assumptions are satisfied.

Diagnostics for Linear Models (OLS)

OLS assumes:

  1. Linearity: Relationship between predictors and outcome is linear.
  2. Homoskedasticity: Errors have constant variance.
  3. Normality: Errors are normally distributed (important for small-sample inference).
  4. Independence: Errors are independent across observations.
  5. No perfect multicollinearity.

Goodness of fit

  • R¬≤: proportion of variance in Y explained by the model.
  • Adjusted R¬≤: penalizes for adding too many predictors.
  • F-test: are all coefficients jointly significant?

üßÆ 2) What is Logit Regression?

Sometimes the outcome is binary (conflict yes/no).
OLS is not appropriate because:

  • Predictions can fall below 0 or above 1
  • Errors are not normally distributed

Logit regression models the probability of conflict:

  • Output is always between 0 and 1
  • Coefficients are in log-odds, not direct changes

Example: Conflict on GDP and Population

Code 
data$conflict_dummy <- ifelse(data$best > 0, 1, 0)

logit1 <- glm(conflict_dummy ~ gdp_pc,
              data = data,
              family = binomial(link = "logit"))

Diagnostics for Generalized Linear Models (GLM, e.g. Logit)

GLMs (like logit) do not assume normality of errors. Instead, they assume:

  1. Correct link function (logit for binary).
  2. Distribution of DV matches family (binomial for binary).
  3. Independence of observations.
  4. No perfect separation (predictors should not perfectly classify outcome).

Goodness of fit

OLS R¬≤ doesn‚Äôt exist for GLMs. Instead:

  • Log-likelihood: higher = better fit.
  • AIC / BIC: lower = better (for model comparison).
  • Pseudo-R¬≤ (McFadden‚Äôs): compares log-likelihood of model vs null model.

üìä 3) Comparing OLS vs Logit

Feature Linear Regression (OLS) Logit Regression
Outcome type Continuous (fatalities, GDP) Binary (conflict yes/no)
Predicted values Any real number Between 0 and 1
Interpretation Change in Y Change in probability (via odds)

Comparing Model Fit Across Models

  • For OLS: Compare R¬≤, Adjusted R¬≤.
  • For GLM: Compare AIC, BIC, Pseudo-R¬≤, AUC.
  • Always balance fit with interpretability and theory.

4) Fixed and Random Effects

What are Fixed Effects (FE) vs Random Effects (RE)?

Panel data: repeated observations for each country across years.

  • Country Fixed Effects (FE) control for all time-invariant country traits (geography, history, culture) by using within-country variation over time.
  • Year Fixed Effects control for shocks common to all countries in a given year (commodity prices, global crises).
  • Two-way FE = country FE + year FE.

Random Effects (RE) assume those unobserved country traits are uncorrelated with the regressors (like GDP). If that assumption is violated, RE is biased; FE stays consistent.

üëâ Rule of thumb: If you suspect correlation between unobserved country characteristics and regressors, use FE. Use RE if the strict exogeneity/orthogonality is plausible and you need time-invariant effects estimated.

Fixed Effects with fixest (recommended)

fixest::feols() uses | to specify fixed effects. It‚Äôs fast and handles multi-way FE and robust SEs easily.

Why clustering?

In panel data (countries √ó years):

  • Observations within a country are likely correlated (serial correlation).
  • Example: if a conflict is ongoing in Sudan in 2012, it‚Äôs likely to persist in 2013.
  • This means residuals are not independent within countries.

If we ignore this, SEs will be too small = p-values too optimistic, making insignificant results look significant.

Clustered SEs adjust the variance‚Äìcovariance matrix of the coefficients to allow:

  • Arbitrary correlation within clusters (e.g., countries).
  • Still assume independence between clusters (e.g., Sudan vs Kenya).

‚úÖ Key Takeaways

  • Use OLS when outcome is continuous (fatalities, GDP, population).
  • Use Logit when outcome is binary (conflict yes/no).
  • Logit ensures predictions stay between 0 and 1.
  • Always think carefully: What is the nature of your dependent variable?
  • FE removes bias from time-invariant unobservables correlated with regressors (within-country identification).
  • RE is efficient if the orthogonality assumption holds; use the Hausman test as a guide.
  • Use two-way FE (country + year) to absorb unit and time shocks.
  • Always report cluster-robust SEs at the panel level (country).
